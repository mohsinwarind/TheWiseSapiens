Rules

1. Each participant must design and code their own portfolio website from scratch. No pre-made templates are allowed.

2. Participants can use frameworks like Bootstrap, React, or any other modern web development frameworks or libraries, but the design and functionality should be their own work.

3. The website must be fully responsive, adapting to different screen sizes and devices.

4. The website should demonstrate good web design principles, such as user-friendly navigation, effective use of color, typography, and imagery.

5. All submissions must be uploaded and hosted on GitHub Pages, and participants must submit the GitHub Pages link through the provided form.

6. Participants are allowed to use AI assistance for guidance but must not directly copy-paste code.

7. Plagiarism or any form of cheating will result in immediate disqualification.

8. Participants must provide proper comments within their code to explain their design decisions and approach.

9. The website must be fully functional and free of major errors or bugs.

10. The judges' decisions are final and binding.

11. Participants should be prepared to present and explain their design choices and code to the judges if required.

12. Feedback and scoring will be provided based on criteria such as design creativity, responsiveness, code efficiency, usability, and overall functionality.

13. The top 3 winners will receive a shoutout on the official page of the organization.

14. Every participant will be awarded a verifiable certificate of participation within 15 days after the deadline.

15. The portfolio should include at least the following sections: About, Skills, Projects, and Contact Information.

16. You must ensure the website is live and accessible for viewing at the time of submission. Broken or inaccessible links will not be accepted.

17. You are encouraged to include animations, interactivity, and custom designs to make the portfolio stand out, but performance and usability should not be compromised.

Frequently Asked Questions

What is the Web Design Challenge?
The Web Design Challenge is a competition where designers and developers can showcase their skills in building portfolio websites using HTML, CSS, and JavaScript, following responsive design principles.
Who can participate in the Web Design Challenge?
Anyone with a passion for web design and development can participate. Whether you're a beginner or experienced, the competition is open to all skill levels.
How can I register for the Web Design Challenge?
You only need to register when submitting your project. The submission link is provided in the Register section on the competition page.
Is there a registration fee?
No, participation in the Web Design Challenge is completely free.
What are the judging criteria?
Judging will be based on design creativity, responsiveness, code quality, usability, and adherence to web development best practices. The use of frameworks like Bootstrap or React is allowed, but originality is key.
Are teams allowed?
This competition is for individual participants only. Team submissions are not permitted.
When does the Web Design Challenge start and end?
The challenge begins on Septemeber 10 and ends on Septemeber 17. Make sure to submit your project before the deadline!
Will participants receive a certificate?
Yes, all participants will receive a certificate of participation within 15 days of the competition’s conclusion.
What frameworks or libraries can I use?
You are free to use any web development framework or library, such as Bootstrap, Tailwind CSS, React, or any others, as long as the design is original and responsive. However, no pre-built templates are allowed.
Do I need to host my project on GitHub?
Yes, you are required to upload your project on GitHub Pages. Make sure to share the link to your live website when submitting your entry.
What if I’ve never hosted a site before?
No worries! We recommend using GitHub Pages for hosting your portfolio. It’s free, easy to set up, and you can find plenty of tutorials online to guide you through the process.
Can I include animations or interactive elements?
Yes, feel free to add animations, transitions, and interactive elements using CSS and JavaScript. Just ensure that they enhance the user experience without compromising performance.
What should my portfolio include?
Your portfolio should include sections like About, Skills, Projects, and Contact Information. You’re encouraged to customize and expand it to reflect your personality and design style.
Where can I get help if I have any questions?
You can contact us via this number 0327-7046356. We will try to respond as quickly as possible.
